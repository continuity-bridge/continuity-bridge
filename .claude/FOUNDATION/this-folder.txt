# FOUNDATION/

## Purpose

**Streamlined rouse system** - Everything an instance needs for fast orientation without loading deep context.

The FOUNDATION directory contains the minimal, essential files for waking up and engaging immediately. Deep context (philosophy, full history, detailed identity) lives in the legacy structure and is loaded on-demand.

## Why FOUNDATION Exists

**Before FOUNDATION:** Wake sequence loaded 5-7k tokens of files before instances could engage.

**After FOUNDATION:** Wake sequence loads ~2.8k tokens (ESSENTIAL.md + RELATIONSHIPS.md), reducing overhead by ~60%.

**Design principle:** Fast rouse first, deep context when needed.

## Relationship to Legacy Structure

**FOUNDATION/** = Fast rouse (getting here)  
**Legacy (identity/, corpus/, memory/)** = Deep context (staying and working)

The legacy structure still exists and serves critical functions:

- **identity/** - Detailed identity files (how-this-was-built.md, etc.)
- **corpus/** - Philosophy, reference materials, Reddit threads
- **memory/** - Session logs, journal, parking-lot, session-index

**When to use each:**
- **FOUNDATION:** Wake sequence, quick reference, essential protocols
- **Legacy:** Deep work, philosophical context, full history

**They coexist permanently.** FOUNDATION doesn't replace legacy - it supplements it with fast-access essentials.
