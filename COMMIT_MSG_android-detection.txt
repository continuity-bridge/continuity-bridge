Fix Android/Termux Detection - Tested on Fire HD 8

Improve capability detection for Android/Termux environments based on
real-world testing on Fire HD 8 (Fire OS 6 / Android 7).

## Problems Fixed

**Platform Detection:**
- OLD: Reported "unknown" on Android/Termux
- NEW: Multi-indicator detection reliably identifies Android

**CLAUDE_HOME Detection:**
- OLD: Couldn't find /sdcard/Claude or /sdcard/Platform_Home
- NEW: Prioritizes Android paths first in candidate list

**Workflow Determination:**
- OLD: Defaulted to TEXT_ONLY despite git being available
- NEW: Recognizes CONTAINER_GIT_WITH_BRIDGE for Android + git

**Bash Context:**
- OLD: Generic "unknown" for Termux
- NEW: Specific "termux_android" context

## Changes Made

### 1. Hard-Code /sdcard/Claude Path

Key insight: /sdcard/ is where adb uploads go and is writeable.

```python
candidates = [
    "/sdcard/Claude",           # Android - MUST BE FIRST
    "/sdcard/Platform_Home",    # Android - alternate
    # ... other platforms
]
```

### 2. Multi-Indicator Android Detection

Termux abstraction layer makes single-check unreliable.
Use ANY of 6 Android indicators:

- /system/bin/app_process (Android system binary)
- /sdcard (external storage)
- /data/data/com.termux (Termux app directory)
- TERMUX_VERSION environment variable
- ANDROID_ROOT environment variable  
- com.termux in PREFIX path

### 3. Android Git Workflow Recognition

```python
if is_android and git_info['available']:
    # Check for work directory
    if has_work_dir or has_bridge:
        workflow = "CONTAINER_GIT_WITH_BRIDGE"
```

### 4. Termux Bash Context

```python
if '/data/data/com.termux' in pwd or 'TERMUX_VERSION' in os.environ:
    return 'termux_android'
```

### 5. Android-Specific Manifest Info

Added android_environment section with:
- Termux version
- sdcard accessibility
- adb upload path
- Recommended clone path
- Setup notes and troubleshooting
